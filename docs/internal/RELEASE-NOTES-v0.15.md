# v0.15.0 - Save Anything, Control Anything & BBcode it

This release is about persistence, control, and polish. **Save Anything**: every stateful pack now carries a `save_state`/`load_state` seam, so any behavior - and any script of your own that adopts the two-method convention - persists itself, in six on-disk formats, with a whole **Save Studio** to preview, export, and generate save support. **Control Anything**: the player-or-AI input seam now covers *every* input-reading pack, so the same behavior can be driven by a human or by your AI without a fork - and pathfinding grows hazards and moving platforms. **BBcode it**: highlight text in a comment and a Discord-style formatting bar floats by the selection, inline and in the dialog. Around the three pillars: StatForge stats, screen and object color tints, rounded sheet corners, an audio last-sound control, and two new in-editor tool guides. The pack count grows from 62 to **70**, everything still compiles to plain typed GDScript with zero runtime dependency, and every pack round-trips byte-for-byte.

## Highlights

- **Save Anything - the save-state seam.** Every stateful pack now ships two plain methods: `save_state() -> Dictionary` (a snapshot of its runtime state, plain data only) and `load_state(state)` (restores it, tolerating missing keys so old saves survive pack updates). No base class, no registration - the Save System duck-types the pair on any node, so **your** scripts join the same convention by implementing the two methods. Eighteen packs are seamed (StatForge, Health, Currency Ledger, SkinVault, Timer, State Machine, Idle Generator, Click Power, Boosts, Upgrades, Prestige, Milestones, Weapon Kit, Storylet Weaver, Loot Table pity, Advanced Random RNG state, ProcRoom run state, Abilities). Put a node in the `persist` group and **Save Game** snapshots it and every seamed behavior child into the slot automatically; **Load Game** restores everything before firing On After Load. Targeted verbs cover a single node, a whole scene-tree group (matched back by node path), or an autoload singleton by name.
- **Six save formats + read helpers.** `config` (ConfigFile, full Variant fidelity - the default), `json` (readable text, every type round-trips exactly), `binary` (compact `store_var`), `csv` (spreadsheet-friendly rows), and now **`ini`** (a portable `[section]` + `key=value` file other tools read) and **`xml`** (structured `<entry>` tags for XML pipelines). All six are type-lossless - an int stays an int, a Vector2 a Vector2 - and honor the encryption key. **Read All** returns the whole slot as one Dictionary, **List Save Keys** enumerates it, and **Read Save File** reads *any* file at a path in a given format without touching the active slot - so importing a backup or a file another tool wrote is one call. **Save File Format** detects a file's format (by extension, then by content), **Save File Is Format** tests it, and **Save Format Is** branches on the active format - so a load flow accepts a dropped-in file without knowing its format up front.
- **The Save Studio** (Tools menu) - one window, three tabs. **Format Preview** runs the real save pipeline on any seamed addon and shows the on-disk output in each format before you commit to one. **Save Slots** browses the project's `user://` saves, views their contents, and exports them anywhere with optional format conversion. **Add Save Support** points at any script, lists its variables (data pre-ticked, node references left off), and generates a copy-paste `save_state`/`load_state` pair - so building save support into your own addons or tools is a three-click job, not a lookup. It is built entirely on a new **Save group in the public `EventSheets` API** (`save_state_code`, `persistable_fields`, `add_save_support`, `has_save_support`, `save_capable_scripts`, `preview_save`), so an extension adds save support as easily as the built-in tooling does. A new **Project Doctor** check flags a behavior or autoload that declares state but ships no seam - advisory, never CI-failing, one-click-fixable in the Studio.
- **Hardened against data loss** (found by an adversarial review of the backend). A slot that exists but cannot be read (bad key, corrupt or truncated file) is no longer silently wiped by the next save - the write is refused. Every write is **atomic** (temp file, then rename over the slot), so a crash mid-save leaves the previous save intact. CSV round-trips backslash values (Windows paths, regex); JSON preserves every integer exactly, not just 64-bit ones (Godot's JSON parser reloads numbers as floats, so ints ride an internal wrapper), keeping RNG determinism intact; and a persist-group entry with no matching node at load time warns instead of vanishing.
- **Control Anything - the player-or-AI input seam, everywhere.** The universal AI drive seam now covers **all** input-reading packs: Car, Tile Movement, Slide Movement, and Virtual Cursor join Platformer Movement, 8-Direction, and the FPS Controller. Flip the exported `ai_controlled` and hold the persistent `ai_*` intents and the pack moves with the same feel knobs the player gets; off (the default) it reads real input exactly as before. Virtual Cursor's new **On Cursor Arrived** trigger makes a fully AI-driven drag & drop a handful of rows (glide to the card, arrive, press, glide to the slot, drop) - verified live end to end with zero real input. The whole convention, including the 3-line recipe for your own addons, is on one page in the new **Player and AI Input** guide.
- **Pathfinding P4: hazards + moving platforms.** **Add Hazard** marks a world rectangle dangerous at routing time (no graph rebuild): deadly hazards block their edges and are never picked as a goal, danger hazards cost 4x so the router detours while a clean route exists. **Add Moving Platform** registers an AnimatableBody2D and the graph gains a PLATFORM edge - the drive runs a full boarding discipline (stop beside the track, board only a platform parked at the boarding side, ride centered, walk off at the far side) so an agent never idles under a descending platform. The **Path Chase** showcase gains a spike zone the routes refuse and an elevator you can ride away from your pursuer.
- **BBcode it - a Discord-style formatting bar in comments.** Highlight text in the Edit Comment dialog and a small unfocused bar floats by the selection: **B / I / U / S** toggle the matching BBCode wrap on exactly the selected text (re-press to unwrap), and the swatch wraps `[color=#hex]`. The selection survives so formats stack, no control steals focus, and `Ctrl+B / I / U` work from the keyboard. The **inline** comment editor got it too - and with it, the custom-drawn inline editor gained text selection itself (Shift+arrows extend, Ctrl+A selects all, a selection deletes or is typed over as one unit). Comments render BBCode in the sheet, so what you format is what you see.
- **Rounding it out.** **StatForge** - stats as a per-node buff stack (add / multiply / override modifiers with tags, sources, and self-expiring timers; threshold rules that fire On Threshold Crossed; whole loadouts as **StatSheetResource** `.tres` grids). **Juice color tints** (2D and 3D) - Set Host Tint blends the host toward a color with a strength dial, Set Screen Tint washes the whole screen (damage red, poison green), Fade Screen Tint eases it out. **Rounded sheet corners** - cells and the event block draw softly rounded, both radii live theme tokens (0 restores the square look). A **last-played-sound** audio control - Play Sound remembers its shot, so Set Last Sound Playback Rate gives every hit natural pitch variation in two rows. The **Rotate** pack (constant spin, 2D or any 3D axis, previewable in the editor), **circular Wrap** arenas, and reusable **drawing prefabs** with the **Draw Lab** showcase. Two new in-editor tool guides - **Using the Save Studio** and **Using the ACE Studio** - a new **documentation index** grouping every guide by task, and every one of the 61 pack guides brought up to 15 worked use cases. Fixed in passing: a pack function with a plain doc comment above its `## @ace_action` block used to be silently dropped from the picker - it now publishes, with its doc comment folded into the description.

**Quality:** every feature landed suite-green with byte-exact round-trip gates and `drifted=0` across all **70 packs**. The save backend is pinned by round-trip tests for all 18 seams (mutate then snapshot then restore then re-snapshot), all six formats (including special characters and empty strings), every read helper, format detection, and the adversarial-review regressions; the AI drag & drop and the pathfinding hazards/elevator are live-verified on camera. Generated code still never depends on the plugin, and emission stays deterministic. Verified on Godot 4.7 stable.
