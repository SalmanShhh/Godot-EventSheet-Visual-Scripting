# v0.14.0 - The Pathfinding & Game-Feel Update

This release gives enemies a brain and games their feel. **Platformer Pathfinding** brings jump-graph navigation to 2D - something Godot has no built-in answer for - and **Nav Agent 3D** wraps the engine's navmesh behind the same verbs, so one vocabulary paths both dimensions. Around them: a complete FPS movement-tech kit, camera juice for 2D *and* 3D, an input overhaul from live Input Map pickers to a working rebind-screen showcase, and an editor that can now preview behaviors without pressing play. The pack count grows from 58 to **62**, everything still compiles to plain typed GDScript with zero runtime dependency, and every pack round-trips byte-for-byte.

## Highlights

- **Platformer Pathfinding (2D jump graphs)** - build a nav graph from your `TileMapLayer` once and `Find Path To`: standable cells become nodes, stairs walk, gaps and ledges connect through JUMP arcs and FALL drops sized to the **sibling PlatformerMovement's real physics** (retune the movement feel and reachability retunes itself). **Portals** link any two spots in one action (doors, teleporters, elevators - the agent blinks through, `On Portal Taken` fires). Patrol discipline ships too: **Ledge Restriction + Leniency** keep a guard on its platform, **strict/relaxed jump positioning**, AI **coyote time**, and **variable jumping** that releases each hop at the height its arc needs. `Find Path To Node` *follows* (auto-repath by interval + movement threshold, `On Repath`), a **stuck watchdog** re-routes agents that stop making progress (`On Waypoint Stuck`), and a **shared Max Paths Per Tick budget** lets twenty chasers repath in waves instead of spiking a frame. See it in the new **Path Chase** showcase: a keyboard player, a chaser that jumps gaps and takes the portal, and a tile bridge that toggles every 3 seconds while the route re-plans live.
- **Nav Agent 3D (navmesh, sheet-shaped)** - the 3D half with **the same verb names**: attach under a `CharacterBody3D` and a `NavigationAgent3D` child is inserted and tuned for you; `Bake Navigation Region` rebakes at runtime; RVO **avoidance** is a toggle; 3D slopes come free from the bake. The FPS Arena gained an orange stalker that paths to you across it.
- **The universal AI drive seam** - PlatformerMovement, 8-Direction, and the FPS Controller all carry the same inert-by-default `ai_controlled` + intent variables, so pathfinding (or your own cutscene/replay driver) steers them with the pack's own acceleration, coyote time, and feel - and with **no** movement pack at all, the pathfinder's built-in fallback driver moves the body itself.
- **FPS Controller movement tech** - **crouching** (the capsule physically shrinks, standing is ceiling-checked - release inside a vent and you pop up at the exit), **crouch sliding**, **wall riding**, and **wall jumping**, each with knobs, state conditions, and triggers, all live in the FPS Arena.
- **Camera juice, both dimensions** - the 2D **Juice** pack gains Recoil, Head Bob, Jitter, and Tilt (all camera effects now compose around one rest pose), and the new **Juice 3D** pack brings trauma Shake, weapon Recoil (cosmetic - it rides on top of mouse look, aim untouched), Head Bob, Jitter, **Lean** (made for wall rides), **FOV Punch**, and aim-down-sights **Zoom FOV To** - applied as additive offsets that never fight whoever owns the camera.
- **The input overhaul** - every action-naming ACE parameter now opens a **live Input Map picker** (your project's actions first, enumerated the moment the dialog opens - and custom ACEs get it with one `input_action` hint); ~30 new **Mouse / Keyboard / InputMap / Gamepad** helper ACEs cover the whole rebind-screen problem (rebind to key/mouse/gamepad in one step, binding-as-text labels, restore defaults, deadzone, gamepad names, capture/release mouse focus, per-event mouse deltas, 3D cursor picking rays); and the new **Input Rebind** showcase is a working options-menu slice - click Rebind, press anything on any device.
- **Preview behaviors in the editor** - select a node and run **Tools > Preview Behaviors on Selected Node**: its attached behaviors animate right in the viewport (Inspector tweaks reshape the motion live) and the node is restored exactly on stop. Any pack opts in with one pure static - the Sine behavior ships it as the reference - and `EventSheets.register_editor_preview` covers scripts you cannot edit.
- **Twelve extension seams in the `EventSheets` API** - row context-menu items, dictionary-defined ACEs, custom parameter editors, Doctor quick-fixes, sheet lifecycle hooks, starter templates, quick-add synonyms, block-kind styling/validation, one-call pack verification, Welcome preferences, custom tours, and palette categories - plus the new **Designing user-friendly ACEs** craft guide distilling how the bundled vocabulary is written.
- **Speaks your language, loads fast** - the editor UI is now fully translatable with **drop-in CSV files** (EFIGS bundled: English, French, Italian, German, Spanish), and plugin boot dropped from ~1.8 s to **~85 ms** via lazy loading.
- **Rounding it out** - the **FPS Controller** pack itself (mouse look, sprint, jump, first/third person switch) with the FPS Arena showcase, System expression/action/condition parity (conversions, degree-based trig, screenshots, seeded randomness, wrap-aware angle tests), and per-showcase folders so demo files never mix.

**Quality:** every feature landed suite-green with byte-exact round-trip gates and `drifted=0` across all 62 packs. The pathfinding and input showcases are **live-verified** - automated runs assert the chaser jumps the gap, takes the portal, and ends beside the player, and the 3D stalker closes distance across the arena - and the graph, A*, ledge filter, budget, capsule-crouch math, and camera-effect integrators are all pinned in tests. Generated code still never depends on the plugin, and emission stays deterministic. Verified on Godot 4.7 stable.
